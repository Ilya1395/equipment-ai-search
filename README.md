# Поиск характеристик моделей оборудования

Streamlit-приложение для поиска характеристик оборудования по открытым источникам и извлечения параметров с помощью бесплатных open-source моделей Hugging Face.

## Что делает приложение

Пользователь вводит:

- `Класс` — классификация верхнего уровня, например `Насосы`;
- `Подкласс` — уточняющая классификация, например `Погружные`;
- `Код модели` — код или модель оборудования, например `ГНОМ 40-25`.

В боковой панели пользователь выбирает поисковик:

- `Google.com`;
- `Ya.ru`;
- `Yahoo.com`.

Программа:

1. формирует несколько поисковых запросов;
2. ищет источники через выбранный поисковик;
3. выдерживает задержку 2 секунды между запросами и загрузкой страниц;
4. открывает найденные страницы;
5. передает найденный текст в выбранную модель Hugging Face;
6. выводит таблицу:
   - `Класс`;
   - `Подкласс`;
   - `Код модели`;
   - `Характеристика`;
   - `Значение`;
   - `Ед. изм.`;
7. дает скачать результат в CSV и XLSX.

### `app.py`

Главный файл Streamlit-приложения. Его нужно указывать в Streamlit Cloud как `Main file path`.

### `utils/search.py`

Поиск через выбранный поисковик. Внутри установлена задержка:

```python
REQUEST_DELAY_SECONDS = 2
```

### `utils/llm.py`

Извлечение характеристик через Hugging Face Inference API и резервное извлечение регулярными выражениями.

### `requirements.txt`

Минимальный набор зависимостей:

```text
streamlit==1.41.1
pandas==2.2.3
requests==2.32.3
beautifulsoup4==4.12.3
openpyxl==3.1.5
huggingface_hub==0.27.1
```

### `runtime.txt`

```text
python-3.11
```

Если Streamlit Cloud все равно использует Python 3.14, выберите Python 3.11 в настройках деплоя Streamlit Cloud.

## Секреты Streamlit

В Streamlit Cloud добавьте секрет:

```toml
HF_TOKEN = "hf_ВАШ_ТОКЕН"
```

Без токена Hugging Face приложение попробует использовать резервное извлечение по шаблонам, но качество будет ниже.

## Запуск в Streamlit Cloud

1. Распакуйте архив.
2. Загрузите содержимое папки `equipment_ai_search` в GitHub-репозиторий.
3. В Streamlit Cloud создайте новое приложение из этого репозитория.
4. В поле `Main file path` укажите:

```text
app.py
```

5. В настройках выберите Python 3.11, если такая настройка доступна.
6. В `Secrets` добавьте `HF_TOKEN`.
7. Нажмите `Deploy`.

## Если источники не находятся

Поисковые системы могут менять разметку или показывать защитную страницу/капчу для облачных серверов. В этом случае попробуйте:

- выбрать другой поисковик в боковой панели;
- уменьшить классификацию до более простых слов, например `Насосы`, `Погружные`;
- проверить код модели, например `ГНОМ 40-25`;
- увеличить количество источников в боковой панели;
- повторить запрос позже.
